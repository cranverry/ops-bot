## 0. 미션

> **박범진이 하루 운영 추적에 쓰는 시간을 2시간 이상 해방시켜, 기획·퀄리티·전략 판단에 집중하게 한다.**

이 문장이 모든 기능 결정의 기준이다.
어떤 기능이든 "이게 박범진의 반복 작업을 제거하는가?"로 P0/P1/P2가 결정된다.

## 1. 비즈니스 진단

### 1.1 수익 구조

```
매출 = 출시 의상 수 × 판매 단가 × 전환율 (하트→구매)
```

### 1.2 현재 실제 병목 (Sprint 5 KR 기준)

| KR | 달성률 | 실제 문제 | OS가 직접 해결하는가 |
|----|--------|-----------|----------------------|
| KR1: 하트 7,000개 | 49% | 마케팅 전환율 | ❌ 직접 해결 불가 |
| KR2: 리터치 10분 이내 8/10 | 미측정 | 생산 품질 | ❌ 직접 해결 불가 |
| KR3: 리깅 효율화 30% | ~80% | 기술 R&D | ❌ 직접 해결 불가 |

### 1.3 그럼에도 OS가 필요한 이유

**현재 구조의 근본 문제:**
박범진 1명이 내부 6명 + 외주 20명+의 실행을 추적하면서 동시에 기획, QC, 전략 판단을 병행한다. 이 구조는 스케일하지 않는다.

```
박범진의 하루 (현재)
──────────────────────────────────────────
운영 추적    │████████████░░░░░░░░░░│ ~40%
기획/QC     │░░░░░░░░████████░░░░░░│ ~30%
전략/마케팅 │░░░░░░░░░░░░░░░░░░░░░░│  ~5%
기타        │░░░░░░░░░░░░░░░░████░░│ ~25%

박범진의 하루 (OS 이후 목표)
──────────────────────────────────────────
운영 추적    │██░░░░░░░░░░░░░░░░░░░░│ ~10%
기획/QC     │░░░░░░░░░░████████████│ ~50%
전략/마케팅 │░░░░░░░░░░░░░░░░░░████│ ~25%
기타        │░░░░░░░░░░░░░░░░░░░░░░│ ~15%
```

OS의 가치: KR을 직접 달성하는 게 아니라, **박범진이 KR을 달성할 시간을 만드는 것.**

### 1.4 대체 가능 vs 대체 불가

| 구분 | 항목 | OS 처리 가능 여부 |
|------|------|-------------------|
| **대체 가능** | 아침 현황 파악 (ClickUp+Notion+Discord 순차 확인) | ✅ 자동 브리핑 |
| | 신규 외주 SOP·브리프 전달 | ✅ 온보딩 자동화 |
| | 버퍼 계산 + 기획 홀드 판단 | ✅ 임계값 자동 실행 |
| | 스프린트 KR 수동 업데이트 | ✅ ClickUp 자동 집계 |
| | 외주 납기 DM 확인 | ✅ 자동 알림 |
| | "이번 주 어디까지 왔어?" 현황 공유 | ✅ 대시보드 비동기 대체 |
| **대체 불가** | 마케팅 전환율 향상 (하트→구매) | ❌ 사람이 판단 |
| | 모델링·리깅 퀄리티 판단 | ❌ 숙련된 인간 |
| | 의상 컨셉 기획 (트렌드 리딩) | ❌ 창의적 감각 |
| | 외주 작가 발굴·관계 유지 | ❌ 인간 관계 |
| | 브랜드 전략 방향 결정 | ❌ 전략적 판단 |
| | 생산 품질의 본질적 향상 | ❌ 기술·훈련의 문제 |

## 2. 제품 정의

### 2.1 구성

```
Spectrum OS
├── OpsBot          자연어 인터페이스 (조회 + 명령)
└── Mission Control 실시간 대시보드 (감시 + 자동화)
```

### 2.2 설계 원칙

| 원칙 | 기준 질문 |
|------|-----------|
| **시간 환원 우선** | 이 기능이 박범진의 반복 작업 몇 분을 제거하는가? |
| **Single Source of Truth** | 데이터는 ClickUp·Notion에 있다. OS는 읽어서 보여준다. 중복 입력 없음 |
| **Action over Report** | 조회로 끝나지 않는다. 명령 실행까지 한 화면에서 완결 |
| **Zero Repetition** | 사람이 두 번 하는 일은 시스템이 한 번만 하게 만든다 |

### 2.3 범위 외

- 외부 고객(구매자) 인터페이스 — 아님
- ClickUp·Notion 대체 — 아님. 이 위에 올라가는 레이어
- 마케팅 자동화, 광고 집행 — 아님 (Booth.pm 외부 플랫폼 영역)

## 3. 사용자

| 역할 | 인물 | 핵심 필요 |
|------|------|-----------|
| L0 Owner | 박범진 | 30초 안에 전체 현황 파악 + 명령 실행 |
| L1 Admin | 최현세, 임주영 등 | 내 유닛 현황 조회 + 팀원 태스크 추적 |
| L2 Member | 한동우, 남태경, 윤건 등 | 오늘 내 할일 + 납품 기준 즉시 확인 |
| L3 External | 프리랜서 20명+ | 내 브리프 + SOP 자기 해결 |


## 4. 시스템 구조

```
┌──────────────────────────────────────────────────────┐
│  OpsBot (GPT-4o mini + Function Calling + SSE)       │
│  → 자연어 질의 응답 (RAG: Notion KB)                 │
│  → 명령 실행 (ClickUp API 직접 제어)                 │
├──────────────────────────────────────────────────────┤
│  Mission Control (React 대시보드)                    │
│  → 실시간 KPI 패널 (5분 폴링)                        │
│  → 버퍼 게이지 + 자동 홀드 실행                     │
├──────────────────────────────────────────────────────┤
│  Spectrum API (Node.js / Vercel Serverless)          │
│  → JWT 인증 (L0~L3)                                  │
│  → Audit Log (모든 명령 실행 기록)                   │
├──────────┬──────────────┬──────────────┬─────────────┤
│ ClickUp  │    Notion    │Google Sheets │  Discord    │
│ 태스크   │  문서/DB/OKR │    재무      │    알림     │
└──────────┴──────────────┴──────────────┴─────────────┘
```

## 5. 기능 명세

> **P0 기준:** 박범진의 하루 20분+ 반복 작업을 제거하는 기능만

### Phase 1 — OpsBot `4주`

#### P0: 운영 조회 (박범진 시간 환원 직결)

| ID | 기능 | 제거되는 반복 작업 | 예상 절감 |
|----|------|-------------------|-----------|
| F-01 | 일일 브리핑 | 아침 ClickUp+Notion+Discord 순차 확인 | 30분/일 |
| F-02 | 버퍼 현황 조회 | 브랜드별 비축량 수동 계산 | 10분/일 |
| F-03 | OKR/KR 진척 조회 | Notion Sprint 페이지 수동 확인 | 15분/주 |
| F-04 | 파이프라인 블로커 조회 | ClickUp 필터 수동 실행 | 10분/일 |
| F-05 | SOP 즉시 조회 | 외주 작가에게 반복 설명 | 30분/건 |

#### P0: 명령 실행 (Action over Report)

| ID | 기능 | 명령 예시 | 실행 |
|----|------|-----------|------|
| F-06 | 태스크 상태 변경 | "[태스크] 리깅 완료 처리" | ClickUp status 업데이트 + Audit Log |
| F-07 | 버퍼 기획 홀드/재개 | "BTQ 기획 홀드" | ClickUp 플래그 + Discord 알림 |
| F-08 | 온보딩 링크 생성 | "모델러 온보딩 링크 만들어줘" | JWT 링크 + L3 브리프 자동 첨부 |

#### P1: 추가 기능 (Phase 1 후반부)

| ID | 기능 | 설명 |
|----|------|------|
| F-09 | 납기 알림 | 마감 24h 전 담당자 Discord DM |
| F-10 | 스프린트 매출 조회 | Google Sheets 연동, RPE 계산 |
| F-11 | 팀원 역할 조회 | "리거 누구야?" → Notion R&R DB |

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **Audit Log** | 모든 명령 실행(F-06~F-08)은 실행자·시각·변경 내용 기록. 조회 가능 |
| **Undo** | F-06 (태스크 상태 변경) 실행 후 5분 이내 취소 가능 |
| **알림 정책** | Discord 알림 최대 5건/시간 묶음 처리. 자동화 알림은 #ops 채널 단일 집중 |
| **응답 시간** | 첫 토큰 < 1.5초. 브리핑 완성 < 5초 |
| **오프라인** | KB 캐시로 SOP 조회 가능. 명령 실행은 온라인 필요 |

## 7. 출시 계획

### 7.1 마일스톤

```
Week 1  Vercel 연결 + RAG 파이프라인 (KB 벡터화)
Week 2  F-01~F-05 구현 (운영 조회 5종)
Week 3  F-06~F-08 구현 (명령 실행 3종) + Audit Log
Week 4  F-09~F-11 + 내부 베타 → Phase 1 완료

Week 5-6   D-01~D-02 (파이프라인 + 버퍼 칸반)
Week 7-8   D-03~D-04 (OKR + 재무)
Week 9-10  D-05~D-06 + OpsBot 연동 완성

Week 11+   A-01~A-05 순차 구현
```

### 7.2 의존성

| # | 항목 | 담당 | 기한 | 블로킹 |
|---|------|------|------|--------|
| D1 | Vercel 계정 연결 | 박범진 | 오늘 | Phase 1 전체 |
| D2 | Google Sheets API 권한 | 박범진 | Week 1 | F-10, D-04 |
| D3 | ClickUp Webhook 설정 | 박범진 | Week 2 | D-01~D-02 실시간 |
| D4 | Discord #ops 채널 ID 확정 | 박범진 | Week 2 | A-01~A-04 |

### 7.3 리스크

| 리스크 | 대응 |
|--------|------|
| ClickUp API Rate Limit | 5분 폴링 + Redis 캐시 |
| 데이터 미입력 (팀 규율) | OS 이전 선행 조건: ClickUp 입력 규율 점검 |
| GPT 비용 초과 | 토큰 제한 설정 + 응답 캐싱 |
