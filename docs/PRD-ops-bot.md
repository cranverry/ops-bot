# PRD: Spectrum OS — 통합 비즈니스 운영 플랫폼
**Product Requirements Document v1.0**
버전: 1.0.0 | 작성일: 2026-03-20 | 작성자: Milo (COO)
상태: **v1.0 확정 — 설계 착수**

---

## Executive Summary

Spectrum은 VRChat 의상 IP를 기반으로 성장하는 크리에이티브 스튜디오다.
현재 인사·재무·생산·OKR·고객관리가 Notion·ClickUp·Discord·스프레드시트에 분산되어 있고,
이 파편화가 의사결정 지연, 반복 소통 비용, 실행 속도 저하를 일으키고 있다.

**Spectrum OS**는 이 모든 운영 영역을 하나의 인터페이스로 통합하는 비즈니스 운영 시스템이다.
자연어 챗봇 인터페이스와 실시간 대시보드를 결합하여, 조직의 모든 실행 판단을 단일 플랫폼 위에서 내릴 수 있게 한다.

**핵심 원칙:**
- **한 번 입력, 전체 반영** — 데이터는 단일 소스에서 흘러야 한다
- **Role-based Visibility** — 역할에 맞는 정보만, 정확하게
- **Automation First** — 사람이 반복하는 일은 시스템이 처리한다

## 1. 배경 및 문제 정의

### 1.1 조직 현황

| 항목 | 현재 상태 |
|------|-----------|
| 브랜드 구조 | OAS·MRG·BTQ·YMY (브랜드 폴더 아카이빙 예정, 유닛 중심으로 재편) |
| 팀 구성 | 내부 6인 + 외부 프리랜서 20인+ |
| 생산 주기 | 2주 스프린트 (분기별 다수 릴리즈) |
| 현재 도구 | Notion (문서) + ClickUp (태스크) + Discord (소통) + 스프레드시트 (재무) |

### 1.2 핵심 문제

| # | 문제 | 비즈니스 임팩트 |
|---|------|----------------|
| P1 | 운영 데이터 파편화 | 현황 파악에 수동 수집 30분+ 소요 |
| P2 | OKR과 실행 단절 | 스프린트 KR 달성률이 일간 의사결정에 미반영 |
| P3 | 재무 가시성 부재 | 브랜드별 RPE, 수익성 실시간 확인 불가 |
| P4 | 반복 소통 비용 | 동일한 SOP·온보딩 정보를 매번 개인이 전달 |
| P5 | 인력 관리 미자동화 | 프리랜서 계약·납기·정산이 수동 추적 |
| P6 | 고객 인사이트 미활용 | VOC·CS 데이터가 기획에 체계적으로 피드백되지 않음 |

## 3. 운영 도메인 및 기능 매트릭스

### 3.1 도메인 정의

Spectrum OS는 6개 운영 도메인을 커버한다.

```
┌──────────┬─────────────────────────────────────────────────────┐
│ 도메인   │ 핵심 기능                                           │
├──────────┼─────────────────────────────────────────────────────┤
│ 인사     │ 팀원 관리, 역할/R&R, 프리랜서 온보딩·계약·정산     │
│ 재무     │ 브랜드별 매출, RPE, 예산 승인, 정산 추적           │
│ 생산운영 │ 파이프라인 10단계, 버퍼 관리, QA 게이트, SOP       │
│ OKR      │ 분기 목표, 스프린트 KR, 체크인, 회고               │
│ 고객관리 │ CS 티켓, VOC 수집, LTV 추적, 플랫폼 메트릭        │
│ 확장전략 │ 신규 브랜드 인큐베이션, 해외 시장, 자동화 로드맵   │
└──────────┴─────────────────────────────────────────────────────┘
```

### 3.2 도메인별 상세 기능

#### 🧑‍💼 도메인 1: 인사 (HR)

**목적:** 내부·외부 인력의 정보와 역할을 중앙 관리하고 온보딩 비용을 제거한다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 팀원 디렉토리 | R&R DB 기반 역할·책임 조회 | P0 |
| 역할 기반 접근 | L0-L3 권한별 정보 노출 제어 | P0 |
| 프리랜서 온보딩 | 자동화된 온보딩 링크 + 브리프 전달 | P0 |
| 계약·납기 추적 | 프리랜서별 작업 할당·마감 모니터링 | P1 |
| 정산 자동화 | 완료 태스크 기반 정산 요청 자동 생성 | P2 |
| 성과 추적 | OKR KR 달성률 개인 연동 | P2 |

**인사 데이터 소스:** Notion R&R DB (`1eb1bca8`) + ClickUp 담당자 필드

#### 💰 도메인 2: 재무 (Finance)

**목적:** 스프린트 단위 매출·비용을 실시간으로 가시화하고 RPE 기반 의사결정을 지원한다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 스프린트 매출 추적 | 브랜드별 2주 단위 하트·매출 집계 | P0 |
| RPE 계산기 | 매출 / 인원(환산) = RPE 실시간 표시 | P0 |
| 예산 승인 플로우 | 박범진 15만원 이하 직접 / 이상 CEO 에스컬레이션 | P1 |
| 외주 비용 추적 | 프리랜서 건당 비용 + 누적 지출 | P1 |
| 수익성 분석 | 브랜드별 margin, 스프린트별 trend | P2 |
| 정산 자동화 | 완료 태스크 → 정산 요청 Draft 자동 생성 | P2 |

**재무 데이터 소스:** Google Sheets (`19coFv9W_bupHMQPJMyL8wwMepESG1HfcMN6f2TU9Jzc`) + ClickUp 태스크

**핵심 지표:**
```
분기 목표 RPE: 280M KRW ÷ 팀 인원
스프린트 매출 목표: 5,449만원 (Sprint 5 기준)
```

#### 🏭 도메인 3: 생산 운영 (Production Ops)

**목적:** 기획→출시 10단계 파이프라인을 자동화하고 버퍼 임계값 기반 기획 홀드를 실행한다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 파이프라인 현황판 | 10단계 진행 상태 실시간 표시 | P0 |
| 버퍼 임계값 관리 | 브랜드별 비축량 ≥ 3 시 기획 홀드 자동 실행 | P0 |
| QA 게이트 | 단계 이동 전 체크리스트 강제 통과 | P0 |
| 태스크 자연어 명령 | "OAS 모델링 완료 처리" → ClickUp 상태 업데이트 | P1 |
| SOP 조회 | 작업 단계별 표준절차서 즉시 조회 | P1 |
| 스탠드업 자동화 | 일일 블로커·진행 요약 자동 수집 | P2 |

**파이프라인 10단계:**

```
기획 → 모델링Pre → 모델링Post → 텍스처Pre → 텍스처Post
→ 리깅 → QA/QC → 출시준비 → 마케팅 → LIVE
```

**버퍼 임계값 로직:**
```
IF buffer[brand] >= 3 → 기획 홀드 + Discord 알림
IF buffer[brand] <= 1 → 기획 재개 권고 + 알림
```

| 회고 자동화 | 스프린트 종료 시 결과 요약 자동 생성 | P1 |

```
KR1 ☪️ : 부스 하트 7,000개 → 현재 6,622 (49%)
KR2 ♉ : 리터치 10분 이내 의상 8/10 달성
KR3 ✡️ : 리깅 효율화 30% 개선 (Blender 내보내기)
```

**OKR 데이터 소스:** Notion CR팀 워크스페이스 (`[개요] Sprint N` 페이지)

```
POST /api/v1/team/onboarding      # 온보딩 링크 생성

# 재무
GET  /api/v1/finance/summary      # 스프린트 매출·RPE (L0)
GET  /api/v1/finance/rpe          # RPE 계산값 (L0)

# 생산운영
GET  /api/v1/pipeline             # 전체 파이프라인 (L1+)
GET  /api/v1/pipeline/:brand      # 브랜드별 (L1+)
POST /api/v1/pipeline/stage       # 스테이지 업데이트
GET  /api/v1/buffer               # 버퍼 현황 전체
POST /api/v1/buffer/hold          # 기획 홀드/재개

# OKR
GET  /api/v1/okr/sprint/:n        # 스프린트 OKR
GET  /api/v1/okr/kr/:id           # KR 진척

# 고객
GET  /api/v1/customer/metrics     # 플랫폼 메트릭
GET  /api/v1/customer/cs          # CS 현황

# 지식
GET  /api/v1/knowledge/search     # RAG 쿼리
POST /api/v1/knowledge/sync       # Notion → KB 동기화 트리거
```
